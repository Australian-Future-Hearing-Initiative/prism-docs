Developer Guide

The PRISM Ecosystem consists of a backend server built with FastAPI and a web frontend built with Flutter. Both rely on Firebase for authentication and database storage. This guide provides instructions on how to set up the PRISM system locally for development or deployment.

Firebase

Get a paid project on Firebase and make a new project.

Unified Setup via Docker

The easiest way to install and run both the PRISM Backend and Frontend is via the prism-setup repository, which uses Docker Compose.

1. Prerequisites

Ensure the following are installed and running: - Docker Desktop - must be installed and running! - Git for Windows, or for Mac and Linux git is pre-installed. - A Firebase project with paid usage enabled. - A Mailgun account or a Gmail account for sending emails.

2. First-time Setup

1. Clone the repository:

Clone this repository and move into it.

git clone https://github.com/Australian-Future-Hearing-Initiative/prism-setup.git
cd prism-setup

2. Configure your keys and secrets:

Create a .env file in the prism-setup directory with the following keys. You can also run the install script to create a template .env file for you.

1. JWT Secret

Generate a unique secret for your session tokens using openssl rand -hex 32 or from https://jwtsecretkeygenerator.com/ the set the value:

JWT_SECRET=xxxx
2. Email Settings

Option A: Mailgun

  1. Sign up for a free account at https://www.mailgun.com/
  2. Follow the instructions to add a domain and verify your email address.
  3. Copy the API key and domain name into your .env file.
MAILGUN_API_KEY=xxxx
MAILGUN_AUTHORIZED_DOMAIN=xxxx.mailgun.org

Option 2. Via Gmail’s SMTP server

To allow PRISM to send emails via Gmail, you need to enable 2-Step Verification and generate an App Password. * Gmail App Password (MAIL_PASSWORD): Google Account → Security & Sign-in → turn 2‑Step Verification ON * Then navigate to your App passwords → create a new app password, give it a name (PRISM) copy the 16‑char code (no spaces) into MAIL_PASSWORD.

Add the following to your .env file:

MAIL_USERNAME=xxxxx@gmail.com
MAIL_FROM=xxxxxx@gmail.com            # must match or be a verified alias
MAIL_PASSWORD=xxxxxxxxxxxxxxxx      # 16‑char Google App Password, NOT EMAIL PASSWORD, see below
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_STARTTLS=True                  # or: MAIL_PORT=465 + MAIL_SSL_TLS=True
MAIL_SSL_TLS=False
MAIL_FROM_NAME=PRISM
USE_CREDENTIALS=True
VALIDATE_CERTS=True
3. Firebase Web API Key

You must have access to a paid Firebase project with an “Editor” or “Owner” role to be able to get a private key. Replace PrismData and prism (web) with the corresponding names of your Firebase project and web app below.

From the Firebase Console, find it in PrismData → Settings → General

Scroll down to Your apps → Web apps → prism (web) → Config (radio button) > apiKey, then add:

FIREBASE_API_KEY=xxxxx
4. Firebase Service Account Key file

From the Firebase Console: PrismData → Settings → Service accounts → Firebase Admin SDK → Generate new private key.

Rename the file to serviceAccountKey.json and place it in the prism-setup directory, then add:

FIREBASE_KEY_PATH=./serviceAccountKey.json

3. Run the installer script:

bash install.sh

This will create the Dockerfiles for the backend and frontend, clone the repos and check for the presence of your keys. If it fails, simply fix the errors as suggested and run the script again.

4. Launch PRISM

Now to launch the frontend and backend services on your local machine run:

docker compose up --build

Wait until setup completes then interact with the app via:

  • Frontend Dashboard: http://localhost:3000
  • Backend API: http://localhost:8000

To stop the frontend and backend services run:

docker compose down

Manual Backend Setup

If you wish to run the FastAPI backend independently without Docker, follow the instructions as detailed in the prism-flutter-backend repository.

1. Set up a virtual environment

Option 1: Using uv (recommended for speed)
  1. Make sure uv is installed:
uv --version

If not installed:

brew install uv             # macOS (Homebrew)
pipx install uv             # cross-platform alternative
  1. Create a virtual environment
uv venv
  1. Activate
source .venv/bin/activate
Option 2: Using venv
  1. Delete the current virtual environment (If you created a virtual environment in another project)
rm -rf venv

or

rm -rf .venv
  1. Create and activate the virtual environment
python -m venv venv

or

uv venv
  1. Activate the virtual environment
source venv/bin/activate  # On Windows: venv\Scripts\activate

Q: Why use a virtual environment? A: It ensures isolated package versions, so dependencies from one project don’t affect another.

2. Add dependencies

Use

uv sync # This will install all dependencies that you need for the project

Tips: How to add new dependencies:

uv add your_dependency

3. Firebase & Environment variables Setup

FIREBASE_KEY_PATH

If you don’t have the access to the Firebase, please contact your firebase admin. For firebase Admin: They need to have “Editor” or “Owner” to be able to get the private key.

  1. In Firebase Console: PrismData → Project settings → Service accounts → Firebase Admin SDK → Generate new private key.
  2. Save the downloaded file as serviceAccountKey.json in this backend repo root.
  3. Add .env by using
  touch .env

Open your .env:

nano .env

and add:

FIREBASE_KEY_PATH=./serviceAccountKey.json
FIREBASE_API_KEY

Find it in Project settings → General → prism (web) -> apiKey, then add:

FIREBASE_API_KEY=YOUR_WEB_API_KEY
JWT_SECRET

Generate a long random string by going to this website: https://jwtsecretkeygenerator.com/

Once you have got the JWT then copy it to here:

JWT_SECRET=YOUR_RANDOM_SECRET
Example .env
FIREBASE_KEY_PATH=./serviceAccountKey.json
FIREBASE_API_KEY=AIza...yourkey
JWT_SECRET=paste_a_long_random_string_here
Email (SMTP) Setup

Add these to your .env to enable outbound email:

Option 1. Via Mailgun: Get your API key from Mailgun API Key page, and setup Mailgun with a domain you can send from.

MAILGUN_API_KEY=your_mailgun_api_key
MAILGUN_AUTHORIZED_DOMAIN=yourdomain.com

Option 2. Via Gmail’s SMTP server

MAIL_USERNAME=your@gmail.com
MAIL_FROM=your@gmail.com            # must match or be a verified alias
MAIL_PASSWORD=xxxxxxxxxxxxxxxx      # 16‑char Google App Password
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_STARTTLS=True                  # or: MAIL_PORT=465 + MAIL_SSL_TLS=True
MAIL_SSL_TLS=False
MAIL_FROM_NAME=PRISM
USE_CREDENTIALS=True
VALIDATE_CERTS=True

MAIL_PASSWORD=xxxxxxxxxxxxxxxx: Google.com -> Google Account → Security → turn 2‑Step Verification ON → App passwords → generate one, copy the 16‑char code (no spaces) into MAIL_PASSWORD.

Troubleshooting (535 BadCredentials): use the app password (not your login), choose exactly one TLS mode (587+STARTTLS or 465+SSL/TLS), restart after .env changes, and ensure MAIL_FROM is allowed.

3. Run the development server

Navigate to the directory where main.py is located, then run:

uv run -- uvicorn app.main:app --reload

or

uvicorn app.main:app --reload

4. Deactivate the virtual environment when you’re done

deactivate

This returns your shell to the global Python environment.

5. View API documentation

Once the server is running, you can explore the API docs at:

  • Swagger UI: http://127.0.0.1:8000/docs
  • ReDoc: http://127.0.0.1:8000/redoc

These documents list all available endpoints and describe how to interact with them.

6. Debug

You should go to the root and run

uv run -- python -m debugpy --listen 127.0.0.1:5678 --wait-for-client \
  -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

Then you can run the debugger.

7. Test

uv run pytest

📁 Project Structure

app/
├── main.py                 # FastAPI application entry point
├── api/
│   └── routes/             # Route definitions (e.g., /experiments, /participants)
├── domain/                 # Pydantic models and schemas
├── services/               # Core business logic (e.g., experiment handling, exports)
├── infrastructure/         # Integrations with Firebase, Firestore, and email
└── utils/                  # Helper functions (e.g., CSV export utilities)

requirements.txt            # Python package dependencies
.env                        # Environment variable configuration (not committed)
README.md                   # Project overview and instructions

This structure follows a modular design that separates concerns across routing, logic, data modeling, and infrastructure. It ensures maintainability and scalability of the backend.

Manual Frontend Setup

If you wish to run the front end Flutter web application without docker, follow the instructions as detailed in the prism-flutter-web-frontend repository.

The PRISM web frontend serves as the interface for data collection and study management for Researchers conducting “experiments” within the PRISM ecosystem as part of the Australian Future Hearing Initiative.

For Researchers

Researchers use this application to design experiments, manage study cohorts, and deploy tasks to participants. It is the control center for your data collection needs.

How to use: 1. Login: Access the researcher dashboard securely. 2. Manage Projects: Create new projects to house specific studies or experiments. 3. Configure Experiments: * Set up various experiment types (e.g., Ecological Momentary Assessment (EMA) surveys, sensor-thresholds). * Define parameters and logical flows for the experiments. 4. Manage Participants: * Add new participants to the system. * Assign participants to specific projects/cohorts. * Monitor participant progress through assigned tasks. * Email participants. * Download audiograms

See the complete userguide: https://docs.google.com/document/d/1_FZVBiN1QqhP50A2izS7KlW_Y021koJM45pSYcHEkMw/edit?tab=t.0#heading=h.qbhdu5cvo4bq

Developer Instructions

1. Launch the Backend Server

This frontend relies on the prism-flutter-backend. See the repo for launch instructions.

2. Configure Environment

Create a .env file in the root directory and add the URL of the backedn server, e.g.:

## Base URL for API calls when running locally
API_BASE_URL=http://127.0.0.1:8000

3. Install FVM (Flutter Version Management)

We use FVM to ensure consistent Flutter versions.

  • Option 1 Dart Pub: dart pub global activate fvm
  • Option 2 Homebrew: brew tap leoafarias/fvm && brew install fvm
  • Option 3 Install Script: curl -fsSL https://fvm.app/install.sh | bash

4. Run the App

Execute the following commands from the root of this repository:

fvm flutter channel stable
fvm flutter upgrade
fvm flutter pub get
## fvm dart run build_runner watch -d    # Optional: watch freezed/json files for changes
fvm flutter run -d chrome             # Run the app in Chrome

🧪 Testing & Architecture

  • Unit & Widget Tests: flutter test

  • Specific Test: flutter test <path/to/test.dart>

  • Integration Tests:

    flutter drive --driver=test_driver/integration_test.dart --target=integration_test/participant_flow_test.dart -d chrome

    Note: Web integration tests inject bytes directly as WebDriver screens cannot handle drag-and-drop.

  • Coverage: flutter test --coverage

Key Directory Structure: * lib/ui: Contains all screens and widgets, separated by feature (e.g., experiment, participant, auth). * lib/domain: Business logic and models. * lib/data: Data repositories and API services.
